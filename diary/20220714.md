맨 처음 진행했던 프로젝트 코드를 오랜만에 수정할 일이 생겼는데 가독성이 굉장히 떨어지게 짜여져 있었음...

일단 하나의 함수가 굉장히 길었음. 너무 중구난방하게 하나의 함수에 전부 박혀있어서 어떤 기능인지 이해하기 위해서 코드를 처음부터 끝까지 다 읽었음

그리고 구조적으로 서비스코드 부분과 data를 읽는 부분으로 구분해서 작성했었는데, 그 부분이 제대로 나눠지지 않아져있었다. 당연히 논리적인 코드는 서비스코드 부분에 있어야했는데, 이번에 봤을때 제대로 나눠지지 않았음. 변명을 해보자면 처음부터 내가 다 한게 아니고 다른 사람이 한걸 넘겨받긴했었는데, 내가 한동안 진행했던 프로젝트이기 때문에 나도 충분히 못했다고 생각함

---

위에서 말한 프로젝트를 오랜만에 읽었던 이유가 업데이트를 중지했기 때문이다. 그래서 해당 프로젝트에서 사용한 모듈들도 예전 버전이었음

그리고 사용한 모듈이 아래같은 형태였음
```
프로젝트 -> A, B 모듈 사용
A모듈 -> B 모듈 사용
```
여기서 해야하는게 A모듈을 업데이트 하는 거였음

A모듈만 업데이트하면 된다고 생각했었는데, A모듈은 최신 버전의 B모듈을 사용하고 있어서 B모듈도 최신버전으로 업데이트 됨
처음에는 A모듈만 업데이트 하는 일은 어렵지 않아서 빨리 끝날꺼라고 생각했었는데, B모듈이 꽤 많이 달라져 있어서 결국 B모듈을 반영하는데 시간이 오래 걸렸음 

이런 일은 어쩔수없다고 생각하긴 하지만 업데이트가 위험한 이유를 오랜만에 겪었음
